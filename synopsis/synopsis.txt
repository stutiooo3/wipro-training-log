// Understanding Promises and Asynchronous JavaScript

1. A Promise is used to handle operations that take time, such as fetching data from a server. It has three states:
* Pending - initial state
* Resolved (Fulfilled) - operation completed successfully
* Rejected - operation failed

2. In the 'getData()' function, a Promise is created using 'new Promise()'. It takes two parameters:
* resolve - called when the task is successful
* reject - called when an error occurs

3. setTimeout()
A built-in function used to delay execution of code.
Runs a function once after a specified time (in milliseconds).
Commonly used to simulate asynchronous tasks (e.g., API calls).

4. Handling Promises
.then()- Executes when the Promise is resolved , Receives the result (data)
.catch()- Executes when the Promise is rejected, Handles errors

5. setInterval()
Executes a function repeatedly at fixed time intervals and continues running until stopped manually.

6. Non-blocking Execution
JavaScript executes asynchronous code without stopping the rest of the program.
Promises help manage async operations in a cleaner and more readable way than callbacks.


 Example: Promise using setTimeout

function getData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve("Data received");
        }, 2000);
    });
}

getData()
    .then(result => console.log(result))
    .catch(error => console.error(error));


* Function returns a Promise
* setTimeout simulates delay (2 seconds)
* resolve() sends success data
* .then() handles success
* .catch() handles errors (not used here since no reject)

7. Timers in JavaScript

* setTimeout() - runs once after delay
* setInterval() - runs repeatedly after fixed interval

Example:

setTimeout(() => console.log("Runs once"), 2000);

setInterval(() => console.log("Repeats every 3 sec"), 3000);

8. Fetch API (Getting Data from Server)

The Fetch API is used to request data from an external source (API).

* fetch() returns a Promise
* Data must be converted into JSON format using .json()

9. Processing API Data

After fetching data, we usually:
1. Convert response to JSON
2. Extract required data
3. Filter specific items
4. Modify data
5. Display results

Example: Fetch + Filter + Map

fetch("https://jsonplaceholder.typicode.com/todos")
    .then(res => res.json())
    .then(data => data.filter(item => item.id % 2 === 0))
    .then(data => data.map(item => ({
        ...item,
        title: "Updated " + item.title
    })))
    .then(data => console.log(data))
    .catch(err => console.error("Error:", err));

-PROTOTYPES and SHARED MEMORY
Adding methods to a constructor's prototype so they are created 
once and shared by all instances.
   - Efficiency: Saves memory by not copying methods to every object instance.
   

- MEMORY MANAGEMENT (STACK VS HEAP)
   Understanding where different types of data are stored in JS.
   - Stack: Stores primitives  and function calls.
   - Heap: Stores complex objects, arrays, and function references.
   - Copying: Primitives are copied by value; Objects are copied by reference.

- THE SPREAD OPERATOR (...)
   Expanding or merging arrays and objects into new structures.

- CALLBACKS & HIGHER ORDER FUNCTIONS 
    Functions that take other functions as arguments (e.g., .forEach()).
   - Callbacks: Executed after another function finishes.
   - Anonymous Callbacks: Used for one-time operations inside methods.

-HOISTING & TEMPORAL DEAD ZONE 
 JavaScript shifts declarations to the top of their scope during compilation.
*var Hoisting: Declaration moves upward, but the value stays undefined until execution reaches it.
*let/const Hoisting: They are also hoisted but stay inside the “Temporal Dead Zone” until initialization, and accessing them before that causes a ReferenceError.
*Function Hoisting: Function declarations are completely hoisted, so they can be called before being written.

-PROTOTYPAL INHERITANCE (OBJECT.CREATE)
 Objects can directly connect to other objects as prototypes.
*Object.create(proto): Generates a new object linked to the given prototype.
*Inheritance: If a property is missing in the child object, JavaScript searches for it in the parent prototype.
*hasOwnProperty(): Checks whether a property belongs directly to the object instead of coming from the prototype chain.

//                                           DAY 8 and DAY 9

1. Default Parameter :
Default Parameter in JavaScript allows us to assign a default value to a function parameter if no value or undefined is passed during the function call.
It helps avoid undefined values and makes the function safer and cleaner.
e.g. :

function greet(name = "Guest") {
    console.log("Hello " + name);
}

greet(); // Hello Guest
greet("stuti"); // Hello stuti

2. Spread Operator (...) : 
The Spread Operator is used to expand elements of an array, object, or iterable into individual elements. 
It is useful for copying arrays, merging arrays, and cloning objects.

e.g. :

const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5];

console.log(arr2); // [1, 2, 3, 4, 5]

3.Rest Operator (...) :
The Rest Operator collects multiple elements and combines them into a single array or object.
 It is mainly used in function parameters and destructuring.

e.g. :

const [first, ...rest] = [10, 20, 30, 40];

console.log(first); // 10
console.log(rest);  // [20, 30, 40]

4. Rest Parameter : 
Rest Parameter allows a function to accept an unlimited number of arguments and stores them in an array. It improves flexibility in function design.

e.g. :

function sum(...numbers) {
    return numbers.reduce((acc, curr) => acc + curr, 0);
}

console.log(sum(10, 20, 30)); // 60

5. Destructuring : 
Destructuring is a JavaScript feature used to extract values from arrays or objects and assign them to variables in a shorter and cleaner way.

e.g. :

const person = {
    name: "stuti",
    age: 22
};

const { name, age } = person;

console.log(name); // stuti
console.log(age);  // 22

6. Position Based Extraction :
 Position Based Extraction is used in Array Destructuring where values are extracted based on their position (index) in the array.

e.g. :

const colors = ["Red", "Blue", "Green"];

const [first, second] = colors;

console.log(first);  // Red
console.log(second); // Blue

7. Key Based Extraction :
 Key Based Extraction is used in Object Destructuring where values are extracted using object property names (keys), not positions.

e.g. :

const student = {
    name: "stuti",
    course: "CSE"
};

const { name, course } = student;

console.log(name);   // stuti
console.log(course); // CSE
Async API : Async API is used to handle operations that take time, such as fetching data from a server, without stopping the execution of the program. It works using async and await, making asynchronous code easier to read and manage than Promises.

e.g. :

async function getData() {
    let response = await fetch("https://api.example.com/data");
    let data = await response.json();
    console.log(data);
}

////////////    TypeScript 
 TypeScript is a superset of JavaScript that adds static typing, interfaces, classes, and better error checking during development. It helps write safer and more maintainable code.

e.g. :

let username: string = "stuti";
let age: number = 22;

1. Merge Interface : Interface merging happens when multiple interfaces with the same name are declared.
 TypeScript automatically combines them into one single interface.

e.g. :

interface User {
    name: string;
}

interface User {
    age: number;
}

const person: User = {
    name: "stuti",
    age: 22
};

2. Declaration Merging :
 Declaration Merging is a feature in TypeScript where multiple declarations with the same name are merged into one definition. It commonly works with interfaces, namespaces, and enums.

e.g. :

interface Student {
    name: string;
}

interface Student {
    course: string;
}

3. Enums :
 Enums (Enumerations) are used to define a set of named constant values. They improve code readability and make values easier to manage.

e.g. :

enum Role {
    Admin,
    User,
    Guest
}

let myRole: Role = Role.Admin;
console.log(myRole); // 0

4. Tuples :
 Tuples are a special type of array in TypeScript where the number of elements and their data types are fixed in a specific order.

e.g. :

let user: [string, number] = ["stuti", 22];

5. Array	Tuple
Stores multiple values of same type	Stores fixed values of different types
Length can change	Length is fixed
Example: number[]	Example: [string, number]

6. Generics : 
Generics allow us to create reusable components that work with different data types without losing type safety.

e.g. :

function identity<T>(value: T): T {
    return value;
}

console.log(identity<string>("Hello"));
console.log(identity<number>(100));
Constraint Generics : Constraint Generics are used when we want to limit the type that a generic can accept using the extends keyword. This ensures only specific types are allowed.

e.g. :

function printLength<T extends { length: number }>(item: T): void {
    console.log(item.length);
}

printLength("Hello");
printLength([1, 2, 3]);
